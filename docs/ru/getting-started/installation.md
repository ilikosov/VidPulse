## Подробное руководство по установке VidPulse

В этом руководстве описаны различные способы установки VidPulse: с помощью Docker, вручную или в производственной среде.

### Способ 1: Установка с помощью Docker (рекомендуется)

#### Требования

- Docker 20.10+
- Docker Compose 2.0+
- 2 ГБ свободной оперативной памяти

#### Шаги установки

1. **Клонирование репозитория**

   ```bash
   git clone https://github.com/your-org/vidpulse.git
   cd vidpulse
   ```

2. **Настройка переменных окружения**

   ```bash
   cp .env.example .env
   ```

   Отредактируйте `.env`, указав необходимые значения:
   - `YOUTUBE_API_KEY`: ваш ключ YouTube Data API v3
   - `DATABASE_URL`: URL базы данных (по умолчанию используется PostgreSQL в контейнере)
   - `REDIS_URL`: URL Redis (по умолчанию используется Redis в контейнере)

3. **Запуск сервисов**

   ```bash
   docker-compose up -d
   ```

4. **Проверка работы**
   ```bash
   docker-compose logs -f app
   ```
   Дождитесь сообщения "Server is running on port 3000".

#### Доступ к сервисам

- Веб-интерфейс: `http://localhost:3000`
- API: `http://localhost:3000/api`
- База данных PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`

### Способ 2: Ручная установка (для разработки)

#### Требования

- Node.js 18+
- PostgreSQL 14+ или SQLite
- Redis 6+

#### Шаги установки

1. **Установка зависимостей**

   ```bash
   npm install
   ```

2. **Настройка базы данных**

   ```bash
   npm run db:migrate
   npm run db:seed
   ```

3. **Запуск сервиса**
   ```bash
   npm run dev
   ```

### Способ 3: Установка в производственной среде

Для производственного развертывания рекомендуется использовать Docker с оркестрацией (Kubernetes) или развертывание на виртуальной машине.

Подробные инструкции по производственному развертыванию см. в разделе [Производственное развертывание](../../operations/deployment/production.md).

### Устранение неполадок

#### Общие проблемы

- **YouTube API ключ не работает**: Убедитесь, что ключ активирован и имеет достаточные квоты.
- **Проблемы с подключением к базе данных**: Проверьте строку подключения в `.env`.
- **Контейнеры не запускаются**: Проверьте логи с помощью `docker-compose logs`.
