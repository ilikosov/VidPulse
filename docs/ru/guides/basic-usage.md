# Базовое использование

_Этот документ находится в разработке. Полная английская версия доступна по ссылке: [English Basic Usage Guide](../../en/guides/basic-usage.md)._

## Основные рабочие процессы и операции

В этом руководстве описаны основные рабочие процессы VidPulse: добавление каналов, синхронизация видео, классификация и управление контентом.

### Добавление YouTube-канала

#### Через административный интерфейс

1. Откройте административный интерфейс по адресу `http://localhost:3000/admin`
2. Перейдите в раздел "Каналы" → "Добавить канал"
3. Введите ID YouTube-канала или URL
4. Настройте параметры синхронизации:
   - Интервал синхронизации
   - Глубина истории (количество видео для backfill)
   - Категории по умолчанию
5. Нажмите "Сохранить"

#### Через API

```bash
curl -X POST http://localhost:3000/api/v1/channels \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "youtubeChannelId": "UC_x5XG1OV2P6uZZ5FSM9Ttw",
    "name": "Google Developers",
    "syncInterval": 60,
    "maxHistoryDays": 30
  }'
```

### Синхронизация видео

#### Автоматическая синхронизация

VidPulse автоматически синхронизирует видео согласно заданному интервалу. Вы можете настроить интервал в конфигурации канала.

#### Ручная синхронизация

```bash
# Синхронизация всех каналов
curl -X POST http://localhost:3000/api/v1/sync/all \
  -H "Authorization: Bearer YOUR_TOKEN"

# Синхронизация конкретного канала
curl -X POST http://localhost:3000/api/v1/sync/channel/CHANNEL_ID \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### Мониторинг синхронизации

Проверьте статус синхронизации в административном интерфейсе или через API:

```bash
curl http://localhost:3000/api/v1/sync/status \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Классификация видео

#### Автоматическая классификация

Новые видео автоматически классифицируются с использованием гибридной системы (правила + LLM).

#### Ручная классификация

1. В административном интерфейсе перейдите к видео
2. Нажмите "Классифицировать"
3. Выберите категории или введите теги
4. Нажмите "Сохранить"

#### Настройка правил классификации

Правила классификации настраиваются в файле `config/classification-rules.json`:

```json
{
  "rules": [
    {
      "name": "tutorials",
      "conditions": [
        { "field": "title", "contains": ["tutorial", "how to", "guide"] },
        { "field": "description", "contains": ["step by step"] }
      ],
      "category": "Educational"
    }
  ]
}
```

### Управление видео

#### Поиск и фильтрация

Используйте административный интерфейс для поиска видео по:

- Названию
- Категории
- Дате публикации
- Статусу классификации

#### Экспорт данных

Экспортируйте видео в различных форматах:

```bash
# Экспорт в CSV
curl http://localhost:3000/api/v1/videos/export?format=csv \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o videos.csv

# Экспорт в JSON
curl http://localhost:3000/api/v1/videos/export?format=json \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o videos.json
```

### Уведомления и вебхуки

Настройте уведомления о новых видео или изменениях статуса:

```bash
curl -X POST http://localhost:3000/api/v1/webhooks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "url": "https://your-service.com/webhook",
    "events": ["video.added", "video.classified"]
  }'
```

### Распространенные сценарии использования

#### Сценарий 1: Мониторинг конкурентов

1. Добавьте каналы конкурентов
2. Настройте ежедневную синхронизацию
3. Используйте классификацию для категоризации контента
4. Экспортируйте отчеты для анализа

#### Сценарий 2: Контент-модерация

1. Настройте правила классификации для выявления нежелательного контента
2. Используйте вебхуки для уведомления модераторов
3. Интегрируйте с системой тикетов

#### Сценарий 3: Исследование контента

1. Соберите исторические данные по каналам
2. Проанализируйте тенденции с помощью экспортированных данных
3. Используйте API для интеграции с аналитическими инструментами
