# 🧩 План разработки проекта (по этапам)

Проект разбит на этапы так, чтобы можно было постепенно собрать работающую систему без перегруза архитектурой.

---

# 🚀 Этап 1 — Базовый каркас системы (MVP foundation)

## Цель:

Поднять минимально работающую систему синхронизации YouTube → база данных.

## Что делаем:

### 🧱 Инфраструктура

- Monorepo (Turborepo + npm workspaces)
- Docker-compose (Postgres + Redis)
- Node.js (Fastify API + Worker)

### 🗄️ База данных

- таблицы:
  - channels
  - videos

### 🔄 Worker v1

- получение videoId через YouTube API
- сохранение в Postgres
- дедупликация по video_id

### 🌐 API v1

- GET /videos
- POST /sync/:channelId

---

## Результат этапа:

✔ видео сохраняются из YouTube  
✔ есть API для просмотра  
✔ есть очередь задач

---

# ⚙️ Этап 2 — Надёжная синхронизация

## Цель:

Сделать систему стабильной и масштабируемой.

## Что добавляем:

### 🔁 Инкрементальная синхронизация

- playlistItems.list
- last synced cursor per channel

### 📦 Batch обработка

- videos.list (batch 50)
- уменьшение количества API вызовов

### 🧠 Idempotency

- UPSERT видео
- защита от дублей

### ⏱ Scheduler (BullMQ repeatable jobs)

- авто-синхронизация каналов

---

## Результат этапа:

✔ система синхронизирует 100+ каналов  
✔ нет дублей  
✔ стабильная очередь
